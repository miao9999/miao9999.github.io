1.3.2 Kotlin 用于 Android

典型的移动应用与典型的企业应用是非常不同的。移动应用是比较小的，它不依赖于整合现有的代码库，它通常需要迅速交付，同时确保在多种设备上可靠地运行。Kotlin 正好可以做这样的项目。

Kotlin 语言的特点是结合支持Android框架的特殊编译器插件，将Android开发变成一个更加高效和愉快的体验。一般的开发任务，例如向控件添加侦听器或将布局元素绑定到字段，都可以用更少的代码完成，甚至有些不用代码（编译器会为你自动生成）。Ando 库（https://github.com/kotlin/anko）也是 Kotlin 团队构建的，通过为许多的 Android API 添加友好的 Kotlin 适配器从而更进一步提高你的体验。

使用 Kotlin 的另一个好处是应用的可靠性更好。如果你有过开发 Android 应用的经验，毫无疑问你非常熟悉提示程序已经停止的这种不幸。当你的程序抛出一个未处理的异常时，通常是空指针异常，这时就会出现这个提示。Kotlin 型系统是能严格的追踪空值，使用空指针异常不那么紧迫。大多数的 Java 代码在 Kotion 中编译时都会导致 空指针异常 ，确保在应用程序到达用户之前解决这个错误。

同归，由于 Kotlin 是完全兼容 Java 6 的，在使用时不会引入任何新的兼容性的问题。你会从 Kotlin 这种新语言的特性中得到好处，并且你依然能够在任何设备上运行你的应用程序，即使不是在最新的 Android 版本。

在性能方面，使用 Kotlin 也不会带来任何坏处。由 Kotlin 编译器生成的代码在执行时也会像常规的 Java 代码一样有效。Kotlin 运行时是是非常简单的，所以不会发生编译后应用包的大小会增加很多。当你使用 lambda 时大多数的 Kotlin 标准库函数会内联它们。内联 lambda 确保没有新的对象被创建，应用程序也不会受到额外GC暂停。
