ViewGroup.LayoutParams

2017-06-30
翻译

Class Overview(类概述)

LayoutParams 是 views 用来告诉它的父容器，它们在父容器中是如何摆放的。参考 ViewGroup Layout Attribute ，列举了该类支持的所有子 view 的属性。

LayoutParams 基类仅仅描述了 view 对宽度和高度的要求。对于每个方面，可以选择以下的一个：

  • FILL_PARENT ：意味着 view 要设置成与它的父容器一样大（减去 padding 值）
  • WRAP_CONTENT：意味着 view 要大到足以包围它的内容（加上 padding 值）
  • 一个确定的数字

因为 ViewGroup 有不同的子类，所以 LayoutParams 也有不同的子类。例如，AbsoluteLayout 有它自己的可以设置 X 和 Y 的值的 LayoutParams 的子类。

Developer Guides(开发者指导)

关于创建用户界面布局的更多信息，请阅读 XML Layout 开发者指导。

XML Attributes(XML 属性)

android:layout_height

指定 view 的基本高度。对于包含布局管理器的任何 View，这是必需的属性。它的值可能是一个固定高度的尺寸值（如『12dp』）或是专用常量之一。

如果是一个尺寸值，它是一个浮点数字加上一个单位如『14.5sp』。可选的单位有：px（pixels 象素），dp（density-independent pixels 密度独立象素），sp（scaled pixels based on perferred font size 基于优选字体大小的缩放像素『一般用于设置字体大小』），mm（millimeters 毫米）。

这也可能是对包含该类型值的资源或主题属性的引用。

可以是以下常量值之一：

Constant(常量) Value(值) Description(描述)
fill_parent -1 该 view 该被设置成与它的父容器一样大（减去 apdding 值）。在 API 8 之后这个常量被 mathc_parent 代替。
match_parent -1 该 view 该被设置成与它的父容器一样大（减去 padding 值）。在 API 8 里有介绍。
wrap_content -1 该 view 要大到足以包围它的内容（加上 padding 值）

这相当于全局属性资源符号layout_height。

android:layout_width

指定 view 的基本宽度。对于包含布局管理器的任何 View，这是必需的属性。它的值可能是一个固定宽度的尺寸值（如『12dp』）或是专用常量之一。

如果是一个尺寸值，它是一个浮点数字加上一个单位如『14.5sp』。可选的单位有：px（pixels 象素），dp（density-independent pixels 密度独立象素），sp（scaled pixels based on perferred font size 基于优选字体大小的缩放像素『一般用于设置字体大小』），mm（millimeters 毫米）。

这也可能是对包含该类型值的资源或主题属性的引用。

可以是以下常量值之一：

Constant(常量) Value(值) Description(描述)
fill_parent -1 该 view 该被设置成与它的父容器一样大（减去 apdding 值）。在 API 8 之后这个常量被 mathc_parent 代替。
match_parent -1 该 view 该被设置成与它的父容器一样大（减去 padding 值）。在 API 8 里有介绍。
wrap_content -1 该 view 要大到足以包围它的内容（加上 padding 值）

这相当于全局属性资源符号layout_width。

Constants(常量)

public static final int FILL_PARENT

View 所要求的高度或宽度的特殊值。FILL_PARENT 意味着 view 要设置成与它的父容器一样大，如果父容器有 padding 值，减去父容器的 padding 值。在 API 8 之后，这个值被 MAHCH_PARENT 代替。

常量的值为：-1（十六进制：0xffffffff）

public static final int MATCH_PARENT

View 所要求的高度或宽度的特殊值。FILL_PARENT 意味着 view 要设置成与它的父容器一样大，如果父容器有 padding 值，减去父容器的 padding 值。在 API 8 里有介绍。

常量的值为：-1（十六进制：0xffffffff）

public static final int WRAP_CONTENT

View 所要求的高度或宽度的特殊值。WRAP_CONTETN 意味着 view 要大到足以包围它的内容，把它自己的 padding 也计算进去。

常量值为：-2（十六进制：0xfffffffe）

Fields(字段)

public int height

view 被设置的高度。可以是 FILL_PARENT(在 API 8 之后被 MATCH_PARENT 代替) 或是 WRAP_CONTENT 常量之一，或是一个具体的大小。

public LayoutAnimationController.AnimationParameters

用在动画布局。

public int width

View 被设置的宽度。可以是 FILL_PARENT(在 API 8 之后被 MATCH_PARENT 代替) 或是 WRAP_CONTENT 常量之一，或是一个具体的大小。

Public Constructors(公共构造方法)

public ViewGroup.LayoutParams(Context c ,AttributeSet attrs)

创建一个新的布局参数集合。这些值是从提供的属性集和 context 中提取的。映射到这组布局参数的XML属性是：

  • layout_width：宽度，可以是一个精确的值，或 WRAP_CONTENT，或是 FILL_PARENT（在 API 8 之后被 MATCH_PARENT 代替）
  • layout_height：高度，可以是一个精确的值，或 WRAP_CONTENT，或是 FILL_PARENT（在 API 8 之后被 MATCH_PARENT 代替）

参数

c 应用的环境

attrs 来自于布局参数值的属性集合

public ViewGruop.LayoutParams(int width,int height)

创建一个指定宽高的布局参数的集合。

参数

width 宽度，可以是一个精确的值，或 WRAP_CONTENT，或是 FILL_PARENT（在 API 8 之后被 MATCH_PARENT 代替）

height 高度，可以是一个精确的值，或 WRAP_CONTENT，或是 FILL_PARENT（在 API 8 之后被 MATCH_PARENT 代替）

public ViewGruop.LayoutParams(ViewGruop.LayoutParams source)

复制构造器。克隆 source 的宽高值。

参数

source 布局参数要复制的值

Public Methods(公共方法)

public void resolveLayoutDirection(int layoutDircetion)

根据布局方向解析布局参数。关心布局方向的子类应该重写这个方法。默认实现什么也不做。

参数

layoutDircetion 布局的方向 LAYOUT_DIRECTION_LIR（从左到右） LAYOUT_DIRECTION_RTL （从右到左）

Protected Methods(被保护的方法)

protected void setBaseAttributes(TypedArray a,int widthAttr, int heightAttr)

从提供的属性里面提取布局参数。

参数

a 提供的属性的风格

widthAttr width 属性的标识符

heightAttr height 属性的标识符
