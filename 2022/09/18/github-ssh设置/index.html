github-ssh设置

2022-09-18

用 git 拉取代码时可以选择 https 也可以选择 ssh 的方式。https 方式简单，但需要输入密码，ssh 方式，可以不输入密码，接下来就配置一下 git 上的 ssh。

git ssh 设置

  1. 先检查电脑上是否已经有 ssh key

    如果有的话，进入第三步。

    如果没有的话，创建一个 ssh key。

  2. 创建 ssh key

    • -t ：指定密钥类型，默认是 rsa，可以省略
    • -C：设置注释文字，比如邮箱
    • -f：指定密钥文件存储文件名，不指定就使用默认文件名，推荐使用默认的文件名

    接下来会生成 id_rsa 和 id_rsa.pub 两个密钥文件。

    然后会输入两次 push 文件时的密码，不输入直接回车，push 的时候就没有密码，直接提交。

    Your identification has been saved in /c/Users/you/.ssh/id_rsa.
    Your public key has been saved in /c/Users/you/.ssh/id_rsa.pub.

​ 当出现上面的提示的时候，就表明 ssh key 已经生成了。

  1. 添加 ssh key 到 github 中

    • 打开 ssh key ，复制文件中内容，注意复制时不要包括 「空格、回车等符号」。

    • 登录 github ，右上角进入 settings，进入 SSH and GPG keys 中，点击 New SSH Key，把刚才复制的 key 粘贴到文本框中，key type 保存默认的 Authentication Key，然后点击 Add SSH Key。

  2. 测试 SSH Key

    打开终端输入

    有提示，输入 yes 回车即可，如果在前面设置了密码，这里会提示输入密码。

    You’ve successfully authenticated, but GitHub does notprovide shell access.

​ 当看到上面的提示后，ssh key 就设置成功了。

​ 拉取代码的时候，通过 ssh 链接。
